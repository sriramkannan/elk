Introduction

This page is to help kickstart one of the basic needs of many technology enterprises - to be the eyes and ears of logs. Most of major programming languages are written with log statements for debugging purposes in a non-development environment. These log messages are primary used or viewed only when there is an issue or to find a story - example: what all pages did a user visit in a given day. To see that information, one has to open the log file residing in a server (in an ocean of servers) or pull it from a central logging server if it was setup in such fashion. A non-technology person cannot effectively view a log and tell a story. To solve that we need something visual. Here's one way to make the logs as visual as possible using all open source technologies - Elasticsearch, logstash and Kibana - in short known widely as ELK stack.

Elasticsearch

Elasticsearch is a search engine written on top of Lucene. It's widely regarded as a distriuted, multi-tenant, NoSql database that stores document based data in JSON format. For the problem we have at hand, we are going to use Elasticsearch for storing parsed log data in JSON format.
Official Elasticsearch Page

Logstash

Logstash is a server-side data processing tool that can take data from multiple sources and can parse/process then and send it out to multiple targets.
Example: log files can be sent from multiple servers to logstash either as a file or on a tcp/udp stream which will process (parsed into JSON) and sent out to a central logging server and/or Elasticsearch.
Official Logstash Page

Kibana

Kibana is a front end dashboard UI that can represent data in multiple formats like pie chart, table, bar graphs, time-series graphs etc. The data requested from Kibana is in the form of a REST API call to Elasticsearch the input of which is provided and configured on the UI.
Official Kibana Page

Installing Elasticsearch

Download the binary from the Elasticsearch download page. Extract the files in a folder of choice. cd to the folder and start the Elasticsearch process in the command line - "./bin/elasticsearch". This should start the process after a few seconds. Simple right? Quick things to know - there is a yml/yaml file in the config folder usually called settings.yml or elasticsearch.yml. This is the only config file that is used to change the default port 9200 which Elasticsearch runs on or the host names etc along with bunch of other administrator level configurations.

Installing Logstash

  1. Download the Logstash binary from the Elasticsearch download page. Extract the files in a folder of choice.
  2. Prepare a config file from scratch - logstash.conf. Remember, logstash's role is only the following three things:
    1. Input
    2. Filter
    3. Output
  3. In this example, we are going to check if logstash is functioning properly. So let's input stdin and output stdout (let's ignore filters for a moment). In other words, we are going to type the input (stdin) manually and see if the output is printed by logstash (stdout) on the terminal. The file looks like this:

    input {
      stdin{}
    }

    ouput {
      stdout{}
    }

  4. Start the logstash process in command line - ./bin/logstash -f logstash.conf
  5. Once the process starts, type something and press enter. Logstash should output the entered message right after.

Installing Kibana

  1. Download the Kibana binary from the Elasticsearch download page. Extract the files in the root folder of a web server (like apache)
  2. Kibana config file is in config/kibana.yml. By default it should point to the local Elasticsearch instance. If you want to connect to a remote ES instance, please feel free to change the host and port number appropriately.
